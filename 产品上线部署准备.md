## ⚠️ 规格告警:必须调整,否则必崩

你选的配置有两处硬伤,不调整部署必失败:

### 1. 内存 1 GiB → 必须升 4 GiB(致命)

本项目要跑 5 个容器,峰值内存远超 1G:

| 容器 | 峰值内存 | 说明 |
|---|---|---|
| backend (gunicorn 4 worker) | **~800 MB** | 启动时 pandas 加载 admission_history.csv (46MB → DataFrame ~300MB) + plans_2025/2026 + 12 省一分一段表 + 135 章程 JSON |
| PostgreSQL 15 | ~200 MB | 默认 shared_buffers + 后台进程 |
| Redis 7 | ~50 MB | 热度追踪 + 缓存 |
| frontend (nginx) | ~30 MB | 静态托管 |
| cron + 系统占用 | ~150 MB | OS + docker daemon |
| **合计** | **~1.3 GB** | **1G 内存必 OOM,后端启动加载数据时直接被 kill** |

方案文档 [部署AI_上线方案.md](file:///Users/sanzhaibanniang/Claude/Projects/gaokao/协作中心/提示词/部署AI_上线方案.md#L19-L20) 明确要求 **2C4G 60G**,这是经过测算的最低规格。

| 维度 | 中国香港 | 新加坡 |
