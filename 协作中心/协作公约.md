# 三方协作公约

> 建立于 2026-06-27 10:01:20
> 参与方：审核员（我）、后端 AI、前端 AI

## 文件夹结构

```
协作中心/
├── 协作公约.md          ← 本文件：协作规则与文件说明
├── 时间线.md            ← 全部沟通的时序流水账（append-only，按时间正序）
├── 审核员_审查报告.md    ← 审核员产出的完整问题清单与证据
├── 后端AI_任务清单.md    ← 派给后端 AI 的任务（含详细修复提示词）
├── 前端AI_任务清单.md    ← 派给前端 AI 的任务（含详细修复提示词）
└── 反馈/                ← 双方 AI 完成后的回复与待审核记录
```

## 角色分工

| 角色 | 职责 |
|------|------|
| 审核员（本账号） | 通读全栈代码、找 bug、产出审查报告、分发任务、验收修复、维护时间线 |
| 后端 AI | 接收 `后端AI_任务清单.md`，按提示词修复后端 Python 代码，完成后在 `反馈/` 写完成说明 |
| 前端 AI | 接收 `前端AI_任务清单.md`，按提示词修复前端 Vue3/TS 代码，完成后在 `反馈/` 写完成说明 |

## 时间线格式

每条记录必须以以下前缀开头，便于追溯"时间顺序"与"对象"：

```
[2026-06-27 10:01:20][审核员] 完成首轮全栈审查，定位 11 个问题，分发任务给两端 AI。
```

- `[时间]`：精确到秒，Asia/Shanghai 时区
- `[对象]`：`审核员` / `后端AI` / `前端AI` / `双方`
- 动作类型用 emoji 标注：📋分发 / ✅完成 / 🔍验收 / ❌驳回 / 💬沟通 / ⚠️阻塞

## 工作流

1. 审核员分发任务 → 时间线写 📋记录，写入对应 `任务清单.md`
2. 某 AI 完成修复 → 在 `反馈/` 新建 `YYYYMMDD_对象_完成说明.md`，时间线写 ✅
3. 审核员验收 → 跑测试/读代码确认，时间线写 🔍通过 或 ❌驳回（附理由）
4. 驳回则回到该 AI 的任务清单顶部加"【返工】"条目

## 约束（来自项目记忆）

- 不影响前端在跑的视觉/交互工作，后端 AI 改动需保证 `uvicorn` 重启后旧接口不破
- 前端 AI 改动需保证 `npm run build` 0 错误
- 修复必须最小化，不引入未要求的重构/抽象
- 支付相关（#1）当前为 mock 模式，但需为上生产堵漏
