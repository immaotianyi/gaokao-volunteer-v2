### 🔴 CRITICAL（3/3 已修复）

#### #C1 · AdvisorChat 多轮上下文丢失 — `history.slice(0, -2)` 截断错误

**文件**：[frontend/components/AdvisorChat.vue](../../frontend/components/AdvisorChat.vue)

**改动**：`advisorStore.history.slice(0, -2)` → `slice(0, -1)`

**根因**：`history` 计算属性已过滤 `streaming` 消息，最后只剩当前 user 消息为最后一个元素。`slice(0, -2)` 会再多删一个，即上一轮 AI 回复被剥离，导致后端看不到自己上一轮的回答，多轮记忆被破坏。

---

#### #C2 · `esc()` 双重转义导致 HTML 实体泄漏

**文件**：
- [frontend/components/AdvisorChat.vue](../../frontend/components/AdvisorChat.vue)（删除 L22 `esc` 函数 + 4 处 `{{ esc(X) }}` → `{{ X }}`）
- [frontend/components/RiskScanner.vue](../../frontend/components/RiskScanner.vue)（删除 L261 `esc` 函数 + 10 处替换）
- [frontend/components/RadarCard.vue](../../frontend/components/RadarCard.vue)（删除 L17 `esc` 函数 + 8 处替换）

**根因**：`esc()` 把 `<` 转成 `&lt;`，模板里再用 `{{ }}` 走 textContent 绑定会把字符串 `"&lt;"` 当字面量渲染，用户看到 `&lt;` 而非 `<`。Vue 的 `{{ }}` 已做一次 textContent 转义，三处 `esc` 函数均应删除。

  const p = form.value
  if (p.score == null || p.score === 0) return "请填写高考总分"
