## 问题清单（11 项）

### #1 支付回调验签可被绕过 — CRITICAL
- **归属**：后端
- **位置**：[backend/services/wechat_pay.py#L222-L225](file:///Users/sanzhaibanniang/Claude/Projects/gaokao/backend/services/wechat_pay.py#L222-L225)
- **问题**：`_verify_signature` 在 `WECHAT_PLATFORM_CERT_PATH` 未配置时直接 `return True`。`/api/pay/notify` 据此跳过验签解密并解锁用户付费报告。当前 mock 模式不触发，但上生产后若漏配置该环境变量，任何人可伪造 `out_trade_no + attach(user_id) + trade_state=SUCCESS` 白嫖解锁，无启动期告警。
- **建议**：`is_configured()` 返回 True 但平台证书未配置时启动日志 WARN；`/api/pay/notify` 在 `ENV=prod` 时拒绝未验签回调。

### #2 advisor 同步调用 Tavily 阻塞事件循环 — MAJOR
- **归属**：后端
- **位置**：[backend/services/advisor.py#L261](file:///Users/sanzhaibanniang/Claude/Projects/gaokao/backend/services/advisor.py#L261)
- **问题**：`maybe_live_search` 是 `async def`，内部直接调同步 `tavily_search`（基于 requests，阻塞 I/O 最长 15s），未用 `asyncio.to_thread` 包装。被 `chat`/`chat_stream` 调用，触发联网搜索即卡死整个事件循环，其他并发请求全部挂起。违反"live_search 必须用 asyncio.gather + to_thread"约束。
- **建议**：`result = await asyncio.to_thread(tavily_search, query + " 高考志愿 2026", 3)`

### #3 邮件发送同步 smtplib 阻塞事件循环 — MAJOR
- **归属**：后端
- **位置**：[backend/services/notification.py#L287-L292](file:///Users/sanzhaibanniang/Claude/Projects/gaokao/backend/services/notification.py#L287-L292)
- **问题**：`send_daily_email` 声明 async 却用同步 `smtplib.SMTP + starttls + login + send_message`，timeout=30，可阻塞事件循环 30s+。
- **建议**：`await asyncio.to_thread(...)` 包装整段 SMTP 逻辑，或改 aiosmtplib。

### #4 差异追踪 new_since_yesterday key 格式不一致 — MAJOR
- **归属**：后端
- **位置**：缓存命中路径 [leakage.py#L198-L199](file:///Users/sanzhaibanniang/Claude/Projects/gaokao/backend/routers/leakage.py#L198-L199) vs 新算路径 [L276-L279](file:///Users/sanzhaibanniang/Claude/Projects/gaokao/backend/routers/leakage.py#L276-L279)
- **问题**：`_track_daily_diff` 在两条路径传入的 `current_keys` 格式不同。缓存命中用 `unique_key` 字段（LeakageOpportunity schema 无此字段，回退为 `university_name+major_name`）；新算路径用 `university_code_group_code_major_name`。导致今日/昨日 key 集合不可比，`new_since_yesterday` 计数失真（可能虚高或恒为 0）。
- **建议**：统一两处 key 构造逻辑，提取为公共函数，确保与存储格式一致。

### #5 heat_tracker Redis 路径 unfav 致收藏数为负 — MAJOR
- **归属**：后端
- **位置**：[backend/services/heat_tracker.py#L57-L60](file:///Users/sanzhaibanniang/Claude/Projects/gaokao/backend/services/heat_tracker.py#L57-L60)
- **问题**：Redis 路径 `unfav` 直接 `pipe.decr(fav_key)`，对不存在 key 从 0→-1（Redis 允许负）；内存路径 L88-89 用 `max(0, ...)` 钳制。`get_heat` 读回 `int(results[1] or 0)` 对 "-1" 返回 -1。前端可能显示"收藏 -1 人"。
- **建议**：Redis 路径加下限保护（Lua 脚本或先 GET 判断）。

### #6 热度追踪 Session 未在异常时关闭 — MAJOR
- **归属**：后端
- **位置**：[backend/routers/leakage.py#L696-L707](file:///Users/sanzhaibanniang/Claude/Projects/gaokao/backend/routers/leakage.py#L696-L707)
- **问题**：`db = SessionLocal()` → `db.add()` → `db.commit()` → `db.close()`，`db.close()` 未放 finally。若 add/commit 抛异常（SQLite locked / 字段超长），跳到 `except: pass`，close 被跳过 → session 与底层连接泄漏，高频下触发 "database is locked"。
- **建议**：改 `with SessionLocal() as db:` 上下文管理器。

### #7 enrollment_kb.query 不返回 rules_text — MAJOR
- **归属**：后端
- **位置**：[backend/services/enrollment_kb.py#L129-L140](file:///Users/sanzhaibanniang/Claude/Projects/gaokao/backend/services/enrollment_kb.py#L129-L140)
- **问题**：`query()` 返回 dict 无 `rules_text` 字段，违反记忆约束"query 必须返回人类可读 rules_text"。当前路由靠单独调 `get_rule_summary()` 绕过，但任何按约束直读 `result["rules_text"]` 的调用方会 KeyError。leakage.py 的 customize 接口（L541 `kb.query`）就是潜在隐患点。
- **建议**：`query()` 返回前补 `result["rules_text"] = self.get_rule_summary(university, major)`。
